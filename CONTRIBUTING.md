# Contributing to ALIGN

Thank you for your interest in contributing to ALIGN! We appreciate all types of
contributions—from bug reports and feature requests to code, tests, or
documentation improvements. To make collaboration smooth, please follow the
steps below.

## Getting started

1. Fork the repository and clone your fork.
2. `cd align` and install dependencies with `npm install`.
3. Create a new branch for your work: `git checkout -b feature/awesome` or
   `bugfix/issue-123`.
4. Make sure your code adheres to existing style and passes any linting.

## Code

- Write clear, concise commit messages. Use [conventional commits
  style](https://www.conventionalcommits.org/).
- Add or update tests for any new functionality and ensure existing tests
  continue to pass.
- Keep changes focused; avoid large refactorings within the same pull request.

## Issues & Pull Requests

- Before opening an issue, search existing issues to avoid duplicates.
- Use the provided issue templates when reporting bugs or requesting features.
- Include steps to reproduce when reporting bugs, and screenshots/logs if
  helpful.
- For pull requests, link to the related issue if there is one.
- Describe what the change does and why it is needed.

## Style

The project uses TypeScript/JavaScript with standard formatting rules. Running
`npm run lint` should highlight any style issues.

## Code of Conduct

By participating in this project, you agree to abide by our [Code of
Conduct](CODE_OF_CONDUCT.md).
